# Markdown 速查表 — 完整语法参考指南

2026 年，Markdown 已经无处不在。**ChatGPT 和 Claude 默认输出 Markdown**。GitHub 用它渲染文档，Slack、Notion、飞书、Discord 全都支持。如果你在网上处理文字，你就在使用 Markdown——无论你是否意识到。

## 基础语法

### 标题

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

### 文本格式

```markdown
**加粗文本**
*斜体文本*
***加粗斜体文本***
~~删除线~~
==高亮文本==（mark 标记）
<ins>下划线</ins>（HTML）
```

### 链接

```markdown
[基础链接](https://example.com)
[带标题的链接](https://example.com "鼠标悬停提示")
<https://自动链接.com>
[引用式链接][ref]

[ref]: https://example.com
```

### 图片

```markdown
![替代文本](image.jpg)
![带标题的图片](image.jpg "图片说明")

![引用式图片][img-ref]
[img-ref]: image.jpg
```

### 引用

```markdown
> 单行引用

> 多行
> 段落引用
>
> —— 作者

> 嵌套
>> 引用
```

### 水平分隔线

```markdown
---
***
___
```

## 列表

### 无序列表

```markdown
- 项目一
- 项目二
  - 嵌套项目（缩进 2 空格）
  - 另一个嵌套
- 项目三
```

也可以混用 `*` 和 `+`：

```markdown
* 第一层
  + 第二层
    - 第三层
```

### 有序列表

```markdown
1. 第一项
2. 第二项
   1. 嵌套有序列表
   2. 另一个嵌套
3. 第三项
```

可以从任意数字开始：

```markdown
57. 从 57 开始
58. 自动续号
```

### 任务列表（GFM）

```markdown
- [ ] 未完成
- [x] 已完成
- [ ] ~~已取消的任务~~
```

效果：

- [ ] 未完成
- [x] 已完成
- [ ] ~~已取消的任务~~

## 表格

```markdown
| 表头 1   | 表头 2    | 表头 3   |
|----------|:--------:|---------:|
| 左对齐   | 居中对齐  | 右对齐   |
| 数据     | 数据     | 数据     |
```

效果：

| 表头 1   | 表头 2    | 表头 3   |
|----------|:--------:|---------:|
| 左对齐   | 居中对齐  | 右对齐   |
| 数据     | 数据     | 数据     |

**对齐方式：**
- `:---` — 左对齐
- `:---:` — 居中对齐
- `---:` — 右对齐

## 扩展语法

### 脚注（GFM）

```markdown
这里是一句带脚注的文字。[^1]

[^1]: 这是脚注的内容。
```

### 删除线（GFM）

```markdown
~~这段文字被划掉了。~~
```

### 表情符号简码

```markdown
:rocket: → 🚀
:fire: → 🔥
:warning: → ⚠️
:white_check_mark: → ✅
:bookmark: → 🔖
```

### 自动链接

```markdown
<https://toolcraftbox.com>
<email@example.com>
```

### 标题锚点 ID

```markdown
## 我的章节 {#custom-id}
```

链接到它：

```markdown
[跳转到章节](#custom-id)
```

### 定义列表

```markdown
术语
: 定义第一行
: 定义第二行
```

### 上标与下标

```markdown
X^2^（上标）
H~2~O（下标）
```

### 转义字符

用反斜杠显示原始 Markdown 字符：

```
\* 不会变成斜体 \*
\# 不是标题
\- 不是列表
\` 不是代码
\[ 不是链接
```

## 常见问题

### Markdown 各平台一样吗？

大体相同，但存在不同**变体**。标准 Markdown 通用。**GFM（GitHub 风格 Markdown）** 增加了表格、任务列表、删除线和脚注。我们的 [Markdown 编辑器](https://toolcraftbox.com/zh/tools/markdown-editor) 支持 GFM。

### Markdown 适合写文档吗？

非常适合。它是 README、技术文档、API 参考和知识库的标准格式。GitHub、GitLab 和大多数静态站点生成器都原生支持 Markdown。
